About Design Patterns

The software engineering has nowadays many years of development with the studies and quality it means. For this reason, many of the problems we can find in the software development nowadays, are already solved by someone else.

Design patterns are those done practices we can use in order to solve some specific problems. The good thing about it is that we don’t have to think or invent a complex solution, you can use a tested and documented one. This will help you writing your code faster and simpler if you use the correct pattern in the correct situation.
